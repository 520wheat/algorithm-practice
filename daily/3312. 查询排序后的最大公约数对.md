# 3312. 查询排序后的最大公约数对

题目链接：[3312. 查询排序后的最大公约数对](https://leetcode.cn/problems/sorted-gcd-pair-queries/description/)

## 思路

**倍数统计 + 容斥原理 + 前缀和 + 二分查找**

如果直接枚举所有数对并计算 `gcd`，时间复杂度是 `O(n²)`，数据量较大时会超时。

本题的核心是：

> 统计每个数作为 `gcd` 恰好出现了多少次，而不是实际生成所有数对。

设数组中最大值为 `maxVal`，我们可以按 `gcd` 的候选值 `g` 进行统计。

## 解题过程

### 1. 统计每个值的出现次数

用哈希表 `cnt` 记录数组中每个值出现的次数，同时记录数组最大值 `maxVal`。

```cpp
unordered_map<int, int> cnt;
int maxVal = 0;
for (int x : nums) {
    cnt[x]++;
    maxVal = max(maxVal, x);
}
```

### 2. 计算 `mul[g]`

`mul[g]` 表示：两个数都能被 `g` 整除的数对数量。

如果数组中能被 `g` 整除的数有 `count` 个，那么这些数之间可以形成：

```cpp
count * (count - 1) / 2
```

个数对。

这些数对的 `gcd` 一定是 `g` 的倍数，但不一定恰好等于 `g`。

例如，两个数都能被 `2` 整除，那么它们的 `gcd` 可能是 `2`，也可能是 `4、6、8...`。

### 3. 计算 `cntG[g]`

`cntG[g]` 表示：`gcd` 恰好等于 `g` 的数对数量。

因为 `mul[g]` 统计的是 `gcd` 为 `g` 的倍数的所有数对，所以：

```cpp
cntG[g] = mul[g] - cntG[2g] - cntG[3g] - cntG[4g] - ...
```

需要从大到小枚举 `g`，这样在计算 `cntG[g]` 时，`cntG[2g]、cntG[3g]...` 都已经计算完成。

### 4. 构建前缀和

将所有 `cntG[g]` 按照 `gcd` 值从小到大累加，得到前缀和数组 `pref`。

其中：

```cpp
pref[i] = {gcd值, gcd <= 当前值的数对总数}
```

这样就可以把排序后的所有 `gcd` 数对映射到连续区间中。

### 5. 处理查询

对于每个查询 `q`，题目要找排序后的第 `q` 个 `gcd` 值。

由于查询是从 `0` 开始计数，所以需要找到第一个满足：

```cpp
pref[i].second > q
```

的位置，该位置对应的 `gcd` 值就是答案。

## 代码

```cpp
class Solution {
public:
    vector<int> gcdValues(vector<int>& nums, vector<long long>& queries) {
        unordered_map<int, int> cnt;
        int maxVal = 0;
        for (int x : nums) {
            cnt[x]++;
            maxVal = max(maxVal, x);
        }

        // 1. 计算 mul[g]：两个数都能被 g 整除的数对数量
        unordered_map<int, long long> mul;
        for (int g = 1; g <= maxVal; g++) {
            long long count = 0;
            for (int x = g; x <= maxVal; x += g) {
                if (cnt.count(x)) {
                    count += cnt[x];
                }
            }
            if (count >= 2) {
                mul[g] = count * (count - 1) / 2;
            }
        }

        // 2. 计算 cntG[g]：gcd 恰好等于 g 的数对数量
        unordered_map<int, long long> cntG;
        for (int g = maxVal; g >= 1; g--) {
            if (!mul.count(g)) continue;

            long long total = mul[g];
            for (int x = 2 * g; x <= maxVal; x += g) {
                if (cntG.count(x)) {
                    total -= cntG[x];
                }
            }
            cntG[g] = total;
        }

        // 3. 构建前缀和
        vector<pair<int, long long>> pref;
        long long sum = 0;
        for (int g = 1; g <= maxVal; g++) {
            if (cntG.count(g)) {
                sum += cntG[g];
                pref.push_back({g, sum});
            }
        }

        // 4. 二分处理查询
        vector<int> ans;
        for (long long q : queries) {
            int l = 0, r = pref.size() - 1;
            while (l < r) {
                int mid = (l + r) / 2;
                if (pref[mid].second > q) {
                    r = mid;
                } else {
                    l = mid + 1;
                }
            }
            ans.push_back(pref[l].first);
        }

        return ans;
    }
};
```

## 示例验证

### 示例 1

```text
nums = [2,3,4]
queries = [0,2]
```

统计出现次数：

```text
cnt: 2 -> 1, 3 -> 1, 4 -> 1
maxVal = 4
```

计算 `mul`：

```text
g = 1: 能被 1 整除的有 3 个，数对数为 3
g = 2: 能被 2 整除的有 2 个，数对数为 1
g = 3: 能被 3 整除的有 1 个，数对数为 0
g = 4: 能被 4 整除的有 1 个，数对数为 0
```

计算 `cntG`：

```text
cntG[2] = mul[2] - cntG[4] = 1
cntG[1] = mul[1] - cntG[2] - cntG[3] - cntG[4] = 3 - 1 = 2
```

构建前缀和：

```text
pref = [(1, 2), (2, 3)]
```

查询：

```text
q = 0， 第一个 pref.second > 0 的 gcd 是 1
q = 2， 第一个 pref.second > 2 的 gcd 是 2
```

答案：

```text
[1, 2]
```

### 示例 2

```text
nums = [4,4,2,1]
queries = [5]
```

统计出现次数：

```text
cnt: 1 -> 1, 2 -> 1, 4 -> 2
maxVal = 4
```

计算 `mul`：

```text
g = 1: 能被 1 整除的有 4 个，数对数为 6
g = 2: 能被 2 整除的有 3 个，数对数为 3
g = 4: 能被 4 整除的有 2 个，数对数为 1
```

计算 `cntG`：

```text
cntG[4] = 1
cntG[2] = mul[2] - cntG[4] = 3 - 1 = 2
cntG[1] = mul[1] - cntG[2] - cntG[4] = 6 - 2 - 1 = 3
```

构建前缀和：

```text
pref = [(1, 3), (2, 5), (4, 6)]
```

查询：

```text
q = 5，第一个 pref.second > 5 的 gcd 是 4
```

答案：

```text
[4]
```

## 复杂度分析

设 `maxVal = max(nums)`，查询数量为 `Q`。

- 时间复杂度：`O(maxVal log maxVal + Q log maxVal)`
  - 倍数统计和容斥过程整体为调和级数复杂度
  - 每个查询使用二分查找
- 空间复杂度：`O(maxVal)`

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 倍数统计 | 用 `mul[g]` 统计 `gcd` 是 `g` 的倍数的数对数量 |
| 容斥原理 | 从大到小减去倍数贡献，得到 `gcd` 恰好等于 `g` 的数量 |
| 前缀和 | 将离散的 `gcd` 值映射到连续区间，方便查询 |
| 二分查找 | 对每个查询找到第一个累计数量大于 `q` 的位置 |
