# 3514. 不同 XOR 三元组的数目 II

题目链接：[3514. 不同 XOR 三元组的数目 II](https://leetcode.cn/problems/number-of-unique-xor-triplets-ii/description/)

## 思路

**集合枚举**

与 [3513. 不同 XOR 三元组的数目 I](3513.%20不同%20XOR%20三元组的数目%20I.md) 不同，本题的 `nums` 不是 `[1, n]` 的排列，而是任意数组。

题目数据范围：

- `n <= 1500`
- `nums[i] <= 1500`

如果直接枚举三元组，会是 `O(n^3)`，会超时。

注意到三元组 XOR 可以拆成两部分：

```text
nums[i] XOR nums[j] XOR nums[k]
= (nums[i] XOR nums[j]) XOR nums[k]
```

因此可以先枚举所有二元 XOR 结果，再和第三个数异或。

## 解题过程

1. 用集合 `two` 存储所有 `nums[i] XOR nums[j]`
2. 枚举所有 `i <= j`，把 `nums[i] XOR nums[j]` 加入 `two`
3. 用集合 `ans` 存储最终三元组 XOR 结果
4. 枚举 `two` 中的每个值 `x`
5. 再枚举每个 `nums[k]`，把 `x XOR nums[k]` 加入 `ans`
6. 返回 `ans.size()`

## 代码

```cpp
class Solution {
public:
    int uniqueXorTriplets(vector<int>& nums) {
        int n = nums.size();
        unordered_set<int> two;
        unordered_set<int> ans;

        for (int i = 0; i < n; i++) {
            for (int j = i; j < n; j++) {
                two.insert(nums[i] ^ nums[j]);
            }
        }

        for (int x : two) {
            for (int k = 0; k < n; k++) {
                ans.insert(x ^ nums[k]);
            }
        }

        return ans.size();
    }
};
```

## 为什么枚举 `i <= j`？

三元组允许下标满足：

```text
0 <= i <= j <= k < n
```

先枚举二元 XOR 时，只需要枚举前两个下标满足 `i <= j` 的情况。

对于固定的 `i` 和 `j`：

```text
nums[i] XOR nums[j]
```

之后再与任意 `nums[k]` 异或，就能得到对应的三元组 XOR 值。

## 为什么不会超时？

### 直接枚举三元组

```text
O(n^3)
```

当 `n = 1500` 时，约为：

```text
1500^3 = 33.75 亿
```

会超时。

### 拆成二元 XOR 后枚举

第一步枚举 `i <= j`：

```text
O(n^2)
```

当 `n = 1500` 时，约为：

```text
1500 * 1500 = 225 万
```

可以接受。

第二步枚举 `two` 和 `nums[k]`。

虽然 `two` 最多看起来可能有 `O(n^2)` 个值，但由于 `nums[i] <= 1500`，XOR 结果的范围很小，实际不同值数量有限，因此也可以通过。

## 示例验证

### 示例 1

```text
nums = [1, 3]
```

先枚举二元 XOR：

```text
1 XOR 1 = 0
1 XOR 3 = 2
3 XOR 3 = 0

two = {0, 2}
```

再枚举第三个数：

```text
0 XOR 1 = 1
0 XOR 3 = 3
2 XOR 1 = 3
2 XOR 3 = 1

ans = {1, 3}
```

答案为：

```text
2
```

## 复杂度分析

- 时间复杂度：`O(n^2 + m * n)`
  - `m` 是 `two` 中不同 XOR 值的数量
  - 在本题数据范围下，可以通过
- 空间复杂度：`O(m + s)`
  - `m` 是二元 XOR 结果数量
  - `s` 是最终三元组 XOR 结果数量

如果只按最朴素的集合规模估计，也可以认为空间复杂度为 `O(n^2)`。

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 先枚举两个数 | 记录所有 `nums[i] XOR nums[j]` |
| 再枚举第三个数 | 得到所有三元组 XOR 值 |
| 用集合去重 | 自动统计不同结果数量 |
| 避免三重循环 | 把 `O(n^3)` 降到可接受范围 |
