# 3513. 不同 XOR 三元组的数目 I

题目链接：[3513. 不同 XOR 三元组的数目 I](https://leetcode.cn/problems/number-of-unique-xor-triplets-i/description/)

## 思路

**数学规律**

`nums` 是 `[1, n]` 的排列，题目要求统计所有三元组：

```text
nums[i] XOR nums[j] XOR nums[k]
```

能得到多少种不同的结果。

因为 `nums` 只是 `[1, n]` 的排列，所以真正影响答案的只有 `n`，和数组顺序无关。

关键结论：

- 当 `n = 1` 时，答案为 `1`
- 当 `n = 2` 时，答案为 `2`
- 当 `n >= 3` 时，答案为大于 `n` 的最小的 `2` 的幂

也就是如果 `n` 的二进制位数为 `bits`，那么答案为：

```text
2^bits
```

## 解题过程

1. 令 `n = nums.size()`
2. 如果 `n <= 2`，直接返回 `n`
3. 否则从 `1` 开始不断乘 `2`，直到得到第一个大于 `n` 的数
4. 返回这个数

## 代码

```cpp
class Solution {
public:
    int uniqueXorTriplets(vector<int>& nums) {
        int n = nums.size();

        if (n <= 2) {
            return n;
        }

        int ans = 1;
        while (ans <= n) {
            ans <<= 1;
        }

        return ans;
    }
};
```

## 为什么 `n <= 2` 要特殊处理？

### `n = 1`

```text
nums = [1]

唯一结果：
1 XOR 1 XOR 1 = 1

不同 XOR 值：{1}
答案 = 1
```

如果套用“大于 `n` 的最小 `2` 的幂”，会得到 `2`，不正确。

### `n = 2`

```text
nums = [1, 2]

可能得到的结果只有：
1 XOR 1 XOR 1 = 1
1 XOR 1 XOR 2 = 2
1 XOR 2 XOR 2 = 1
2 XOR 2 XOR 2 = 2

不同 XOR 值：{1, 2}
答案 = 2
```

如果套用“大于 `n` 的最小 `2` 的幂”，会得到 `4`，不正确。

## 为什么 `n >= 3` 时答案是 `2` 的幂？

设 `n` 的二进制位数为 `bits`。

由于参与 XOR 的数都在 `[1, n]` 中，所以每个数最多只有 `bits` 位。因此任意三个数异或后的结果也不会超过：

```text
2^bits - 1
```

也就是说，所有结果最多只能落在：

```text
[0, 2^bits - 1]
```

这个范围中，总共有 `2^bits` 个可能值。

当 `n >= 3` 时，可以构造出这个范围内的所有值，所以答案就是：

```text
2^bits
```

而 `2^bits` 正好就是大于 `n` 的最小的 `2` 的幂。

## 示例验证

### 示例 1

```text
nums = [1, 2]
n = 2

n <= 2，直接返回 2
```

### 示例 2

```text
nums = [3, 1, 2]
n = 3

ans = 1
ans <= 3，ans = 2
ans <= 3，ans = 4

返回 4
```

可以得到的不同结果为：

```text
{0, 1, 2, 3}
```

### 额外测试

```text
nums = [1, 5, 4, 2, 3]
n = 5

ans = 1
ans <= 5，ans = 2
ans <= 5，ans = 4
ans <= 5，ans = 8

返回 8
```

## 复杂度分析

- 时间复杂度：`O(1)`
  - `n` 的范围有限，循环次数只和整数二进制位数有关
- 空间复杂度：`O(1)`
  - 只使用常数个变量

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| `n <= 2` | 特殊情况，答案就是 `n` |
| `n >= 3` | 答案为大于 `n` 的最小 `2` 的幂 |
| 数组顺序 | `nums` 是 `[1, n]` 的排列，顺序不影响答案 |
| XOR 上界 | 结果最多有 `n` 的二进制位数那么多位 |
