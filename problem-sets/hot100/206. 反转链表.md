# 206. 反转链表

题目链接：[206. 反转链表](https://leetcode.cn/problems/reverse-linked-list/description/)

## 思路

**迭代（双指针）**

遍历链表，逐个改变节点的 `next` 指针方向，让它指向前一个节点。

反转过程中需要维护两个指针：

- `prev`：当前节点反转后要指向的前一个节点
- `cur`：当前正在处理的节点

同时还需要临时保存 `cur->next`，否则修改 `cur->next` 后会导致后续链表断开，无法继续遍历。

## 解题过程

1. 初始化 `prev = nullptr`，`cur = head`
2. 当 `cur != nullptr` 时循环：
   - 保存 `next = cur->next`，防止断链
   - 令 `cur->next = prev`，反转当前节点指针方向
   - `prev = cur`，`prev` 前移
   - `cur = next`，`cur` 前移
3. 循环结束后，`prev` 指向新的头节点
4. 返回 `prev`

## 代码

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;

        while (cur != nullptr) {
            ListNode* next = cur->next;
            cur->next = prev;
            prev = cur;
            cur = next;
        }

        return prev;
    }
};
```

### 示例 1

```text

```

执行过程：

```text

next = 2, 1 → null, prev = 1, cur = 2

cur = 2, prev = 1
next = 3, 2 → 1, prev = 2, cur = 3

cur = 3, prev = 2
next = 4, 3 → 2, prev = 3, cur = 4

cur = 4, prev = 3
next = 5, 4 → 3, prev = 4, cur = 5

cur = 5, prev = 4

```

返回：

```text

```

## 递归写法

递归函数的含义是：**把以 `head` 为头的链表反转，并返回反转后的新头节点**。

```text
reverseList(head) = 反转以 head 开头的链表，返回新头节点
```

递归写法的核心是：

- 先反转 `head->next` 后面的链表
- 再把 `head` 接到反转后链表的末尾

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        if (head == nullptr || head->next == nullptr) {
            return head;
        }

        ListNode* newHead = reverseList(head->next);
        head->next->next = head;
        head->next = nullptr;

        return newHead;
    }
};
```

### 递归过程

以链表 `1 → 2 → 3 → 4 → 5` 为例。

#### 第 1 步：递归到底

```cpp
if (head == nullptr || head->next == nullptr) {
    return head;
}
```

递归会不断深入，直到最后一个节点：

```text
reverseList(1)
  → reverseList(2)
    → reverseList(3)
      → reverseList(4)
        → reverseList(5)
          → head->next == nullptr，返回 5
```

此时 `5` 就是反转后链表的新头节点，也就是 `newHead`。

#### 第 2 步：开始回溯

当 `reverseList(4)` 收到 `newHead = 5` 时：

```cpp
// head = 4, newHead = 5
head->next->next = head;
head->next = nullptr;
return newHead;
```

图解：

```text
反转前：4 → 5 → nullptr

执行 head->next->next = head：
4 → 5 → 4    // 临时形成环

执行 head->next = nullptr：
5 → 4 → nullptr
```

继续回溯到 `reverseList(3)`，此时 `newHead` 仍然是 `5`：

```cpp
// head = 3, newHead = 5
head->next->next = head;  // 让 4 的 next 指向 3
head->next = nullptr;     // 让 3 的 next 断开
return newHead;           // 继续返回 5
```

图解：

```text
当前要处理：3 → 4
已经反转的后半部分：5 → 4 → nullptr

执行 head->next->next = head：
5 → 4 → 3

执行 head->next = nullptr：
5 → 4 → 3 → nullptr
```

以此类推，最终得到：

```text
5 → 4 → 3 → 2 → 1 → nullptr
```

### 完整执行过程

```text
递归深入：
reverseList(1)
  reverseList(2)
    reverseList(3)
      reverseList(4)
        reverseList(5) 返回 5

回溯：
5 → nullptr

4 → 5 → nullptr
5 → 4 → nullptr，返回 5

3 → 4 → 5 → nullptr
5 → 4 → 3 → nullptr，返回 5

2 → 3 → 4 → 5 → nullptr
5 → 4 → 3 → 2 → nullptr，返回 5

1 → 2 → 3 → 4 → 5 → nullptr
5 → 4 → 3 → 2 → 1 → nullptr，返回 5
```

### 代码逐行解释

```cpp
ListNode* reverseList(ListNode* head) {
    // 递归终止条件：空链表或只有一个节点
    if (head == nullptr || head->next == nullptr) {
        return head;
    }

    // 递归反转 head->next 开头的链表
    ListNode* newHead = reverseList(head->next);

    // head->next 原本是 head 后面的节点
    // 反转后，head->next 变成了反转后链表的尾节点
    // 所以让 head->next->next = head，把 head 接到末尾
    head->next->next = head;

    // 断开原来的 next，防止形成环
    head->next = nullptr;

    // 返回反转后的新头节点
    return newHead;
}
```

### 小例子：`1 → 2 → 3`

```text
调用 reverseList(1)
  ↓
调用 reverseList(2)
  ↓
调用 reverseList(3)
  ↓
head = 3, head->next = nullptr，返回 3

回到 reverseList(2)：
head = 2, newHead = 3
执行 head->next->next = head，相当于 3->next = 2
执行 head->next = nullptr，相当于 2->next = nullptr
返回 3

回到 reverseList(1)：
head = 1, newHead = 3
执行 head->next->next = head，相当于 2->next = 1
执行 head->next = nullptr，相当于 1->next = nullptr
返回 3
```

最终结果：

```text
3 → 2 → 1 → nullptr
```

关键理解：回溯时，`head->next` 指向的是**反转后的后半部分链表的尾节点**。所以 `head->next->next = head` 就是在把 `head` 接到反转后链表的末尾。

递归写法更简洁，但会使用递归调用栈。

## 图解

```text
反转前：1 → 2 → 3 → nullptr
反转后：nullptr ← 1 ← 2 ← 3
```

迭代过程：

```text
prev    cur    next
null    1      2
1       2      3
2       3      null
3       null   -
```

## 复杂度分析

- 时间复杂度：`O(n)`
  - 每个节点只会被访问一次
- 空间复杂度：`O(1)`
  - 迭代写法只使用常数个指针变量

如果使用递归写法，空间复杂度为 `O(n)`，来自递归调用栈。

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 保存 `next` | 防止修改指针后断链 |
| `cur->next = prev` | 反转当前节点指针方向 |
| `prev = cur` | 让 `prev` 前移到当前节点 |
| `cur = next` | 继续处理原链表中的下一个节点 |
| 返回 `prev` | `prev` 是反转后的新头节点 |
