# 76. 最小覆盖子串

题目链接：[76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/description/)

## 思路

**滑动窗口 + 哈希表**

用两个指针 `l` 和 `r` 维护一个窗口，不断扩展 `r`，直到窗口包含 `t` 中所有字符，然后收缩 `l` 寻找最小窗口。

- `need`：记录 `t` 中每个字符需要的数量
- `have`：记录当前窗口中每个字符的数量
- `cnt`：记录窗口中已经满足需求的字符个数

## 解题过程

1. 统计 `t` 中每个字符出现次数，存入 `need`
2. 遍历右指针 `r`：
   - 将 `s[r]` 加入窗口，更新 `have`
   - 如果当前字符满足需求，`cnt++`
   - 当 `cnt == t.size()` 时，窗口已包含所有字符：
     - 更新最小窗口
     - 收缩左边界，直到窗口不再满足条件
3. 返回最小窗口子串

## 代码

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, have;
        for (char c : t) need[c]++;

        int minLen = INT_MAX, start = 0;

        for (int r = 0, l = 0, cnt = 0; r < s.size(); r++) {
            char c = s[r];
            have[c]++;

            if (have[c] <= need[c]) cnt++;

            while (cnt == t.size()) {
                int len = r - l + 1;
                if (len < minLen) {
                    minLen = len;
                    start = l;
                }

                have[s[l]]--;
                if (have[s[l]] < need[s[l]]) cnt--;
                l++;
            }
        }

        return minLen == INT_MAX ? "" : s.substr(start, minLen);
    }
};
```

## 示例验证

### 示例 1

```text
s = "ADOBECODEBANC", t = "ABC"

need: A:1, B:1, C:1

r=0, 'A': have[A]=1≤1 → cnt=1
r=1, 'D': have[D]=1, need[D]=0 → cnt不变
r=2, 'O': 跳过
r=3, 'B': have[B]=1≤1 → cnt=2
r=4, 'E': 跳过
r=5, 'C': have[C]=1≤1 → cnt=3 → 窗口 "ADOBEC"

while cnt==3:
  len=6, minLen=6, start=0
  收缩: have[A]=0 < need[A]=1 → cnt=2, l=1
cnt=2 退出循环

继续扩展...
最终找到 "BANC" 长度4
```

返回 `"BANC"`。

### 示例 2

```text
s = "a", t = "a"

need: a:1

r=0, 'a': have[a]=1≤1 → cnt=1
while cnt==1:
  len=1, minLen=1, start=0
  收缩: have[a]=0 < need[a]=1 → cnt=0, l=1

返回 "a"
```

### 示例 3

```text
s = "a", t = "aa"

need: a:2

r=0, 'a': have[a]=1≤2 → cnt=1
cnt=1 != t.size()=2 → 不进 while

返回 ""
```

## 复杂度分析

- 时间复杂度：`O(m + n)`
  - `m` 是 `s` 的长度，`n` 是 `t` 的长度
  - 每个指针最多移动一次
- 空间复杂度：`O(1)`
  - 这里字符集大小是固定的，哈希表空间可视为常数

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| `cnt` | 记录满足需求的字符个数，不是窗口长度 |
| 贡献判断 | `have[c] <= need[c]` 时说明当前字符有贡献 |
| 收缩条件 | `cnt == t.size()` 说明窗口有效，尝试收缩 |
| 失去贡献 | `have[s[l]] < need[s[l]]` 时 `cnt--` |
