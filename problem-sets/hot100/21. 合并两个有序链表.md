# 21. 合并两个有序链表

题目链接：[21. 合并两个有序链表](https://leetcode.cn/problems/merge-two-sorted-lists/description/)

## 思路

**迭代（双指针）**

两个链表本身都是升序的，因此可以用两个指针分别遍历 `list1` 和 `list2`。

每次比较两个指针当前节点的值，把较小的节点接到新链表后面，然后对应指针后移。

当其中一个链表遍历完后，另一个链表剩下的部分已经是有序的，可以直接接到结果链表末尾。

## 解题过程

1. 创建哑节点 `dummy`，用来简化头节点的处理
2. 用 `cur` 指向结果链表当前末尾
3. 当 `list1` 和 `list2` 都不为空时：
   - 如果 `list1->val <= list2->val`，将 `list1` 当前节点接到 `cur` 后面
   - 否则将 `list2` 当前节点接到 `cur` 后面
   - 被接入的链表指针后移
   - `cur` 后移
4. 循环结束后，将剩余的非空链表直接接到 `cur` 后面
5. 返回 `dummy.next`

## 代码

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        ListNode dummy;
        ListNode* cur = &dummy;

        while (list1 != nullptr && list2 != nullptr) {
            if (list1->val <= list2->val) {
                cur->next = list1;
                list1 = list1->next;
            } else {
                cur->next = list2;
                list2 = list2->next;
            }
            cur = cur->next;
        }

        cur->next = (list1 != nullptr) ? list1 : list2;

        return dummy.next;
    }
};
```

## 示例验证

### 示例 1

```text
list1: 1 → 2 → 4
list2: 1 → 3 → 4
```

合并过程：

```text
比较 1 和 1：1 <= 1，接 list1 的 1
比较 2 和 1：2 > 1，接 list2 的 1
比较 2 和 3：2 <= 3，接 list1 的 2
比较 4 和 3：4 > 3，接 list2 的 3
比较 4 和 4：4 <= 4，接 list1 的 4

list1 为空，直接接上 list2 剩余的 4
```

结果：

```text
1 → 1 → 2 → 3 → 4 → 4
```

### 示例 2

```text
list1: 空
list2: 空

两个链表都为空，返回空链表
```

### 示例 3

```text
list1: 空
list2: 0

list1 为空，直接返回 list2
结果：0
```

## 递归写法

也可以用递归来写：

- 如果某个链表为空，直接返回另一个链表
- 否则比较两个头节点
- 较小的节点作为当前结果节点
- 它的 `next` 指向递归合并后的链表

```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        if (list1 == nullptr) {
            return list2;
        }
        if (list2 == nullptr) {
            return list1;
        }

        if (list1->val <= list2->val) {
            list1->next = mergeTwoLists(list1->next, list2);
            return list1;
        } else {
            list2->next = mergeTwoLists(list1, list2->next);
            return list2;
        }
    }
};
```

递归写法更简洁，但会使用递归调用栈。

## 复杂度分析

- 时间复杂度：`O(m + n)`
  - `m` 和 `n` 分别是两个链表的长度，每个节点最多访问一次
- 空间复杂度：`O(1)`
  - 迭代写法只使用常数个指针变量

如果使用递归写法，空间复杂度为 `O(m + n)`，来自递归调用栈。

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 哑节点 | 简化结果链表头节点处理 |
| 双指针 | 分别遍历两个有序链表 |
| 比较取小 | 每次把较小节点接到结果链表后面 |
| 接剩余 | 一个链表为空后，直接接上另一个链表 |
| 返回值 | 返回 `dummy.next` |
