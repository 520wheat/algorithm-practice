# 169. 多数元素

题目链接：[169. 多数元素](https://leetcode.cn/problems/majority-element/description/)

## 思路

**摩尔投票法（Boyer-Moore Voting Algorithm）**

核心思想是：不同元素两两抵消，多数元素最终一定会剩下。

题目保证多数元素一定存在，并且它出现的次数严格大于 `n / 2`。因此即使把多数元素和非多数元素一一配对抵消，最后剩下的也一定是多数元素。

维护两个变量：

- `major`：当前候选的多数元素
- `cnt`：当前候选元素的票数

遍历数组时：

1. 如果 `cnt == 0`，说明之前的候选人已经被抵消完，当前元素成为新的候选人
2. 如果当前元素等于 `major`，说明是赞成票，`cnt++`
3. 如果当前元素不等于 `major`，说明是反对票，`cnt--`，抵消一个候选元素

由于真正的多数元素数量超过一半，经过不断抵消后，最终留下的候选人一定是它。

## 解题过程

1. 初始化 `major = 0`，`cnt = 0`
2. 遍历数组中的每个元素 `x`：
   - 如果 `cnt == 0`，更新 `major = x`
   - 如果 `x == major`，令 `cnt++`
   - 否则令 `cnt--`
3. 遍历结束后，`major` 就是多数元素

## 代码

```cpp
class Solution {
public:
    int majorityElement(vector<int>& nums) {
        int major = 0;
        int cnt = 0;

        for (int x : nums) {
            if (cnt == 0) {
                major = x;
            }

            if (major == x) {
                cnt++;
            } else {
                cnt--;
            }
        }

        return major;
    }
};
```

## 示例验证

### 示例 1

```text
nums = [3, 2, 3]

x = 3：cnt = 0，major = 3，major == x，cnt = 1
x = 2：major != x，cnt = 0
x = 3：cnt = 0，major = 3，major == x，cnt = 1

返回 3
```

### 示例 2

```text
nums = [2, 2, 1, 1, 1, 2, 2]

x = 2：cnt = 0，major = 2，cnt = 1
x = 2：major == x，cnt = 2
x = 1：major != x，cnt = 1
x = 1：major != x，cnt = 0
x = 1：cnt = 0，major = 1，cnt = 1
x = 2：major != x，cnt = 0
x = 2：cnt = 0，major = 2，cnt = 1

返回 2
```

## 图解：两两抵消

```text
数组：[2, 2, 1, 1, 1, 2, 2]

2 和 1 抵消 → [2, 1, 1, 2, 2]
2 和 1 抵消 → [1, 2, 2]
1 和 2 抵消 → [2]

最终剩下 2
```

## 复杂度分析

- 时间复杂度：`O(n)`
  - 只需要遍历数组一次
- 空间复杂度：`O(1)`
  - 只使用常数个变量

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| `cnt == 0` | 当前候选人被抵消完，新候选人上位 |
| `major == x` | 赞成票，票数加一 |
| `major != x` | 反对票，票数减一，抵消一对不同元素 |
| 正确性保证 | 多数元素数量大于 `n / 2`，抵消后必然剩下 |
