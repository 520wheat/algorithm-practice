# 287. 寻找重复数

题目链接：[287. 寻找重复数](https://leetcode.cn/problems/find-the-duplicate-number/description/)

## 思路

**快慢指针（Floyd 判圈算法）**

可以把数组看成一条链表：

```text
索引 i 指向 nums[i]
```

也就是每个位置 `i` 的下一个位置是 `nums[i]`。

题目给定数组长度为 `n + 1`，而元素值范围是 `[1, n]`。这意味着：

- 每个 `nums[i]` 都可以作为合法下标继续跳转
- 因为有 `n + 1` 个位置，却只有 `n` 种取值，所以一定存在重复数字
- 重复数字会导致链表中出现环

环的入口就是重复的数字。

因此问题可以转化为：在链表中找到环的入口。

使用 Floyd 判圈算法：

1. 快指针一次走两步，慢指针一次走一步，先找到环中的相遇点
2. 再让一个指针从起点出发，另一个指针从相遇点出发，两者每次都走一步
3. 它们再次相遇的位置就是环的入口，也就是重复数

## 解题过程

1. 初始化：
   - `slow = nums[0]`
   - `fast = nums[nums[0]]`
2. 快慢指针找相遇点：
   - `slow = nums[slow]`
   - `fast = nums[nums[fast]]`
   - 直到 `slow == fast`
3. 找环入口：
   - `ptr` 从下标 `0` 出发
   - `slow` 从相遇点出发
   - 两个指针每次都走一步
   - 再次相遇时，该位置就是重复数字
4. 返回相遇位置

## 代码

```cpp
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int slow = nums[0];
        int fast = nums[nums[0]];

        while (slow != fast) {
            slow = nums[slow];
            fast = nums[nums[fast]];
        }

        int ptr = 0;
        while (ptr != slow) {
            ptr = nums[ptr];
            slow = nums[slow];
        }

        return ptr;
    }
};
```

## 示例验证

### 示例 1

```text
nums = [1, 3, 4, 2, 2]

数组下标： 0  1  2  3  4
数组元素： 1  3  4  2  2

链表：
0 → 1 → 3 → 2 → 4
            ↑   ↓
            └───┘

环的入口 = 2
```

快慢指针找相遇点：

```text
初始：slow = nums[0] = 1，fast = nums[nums[0]] = nums[1] = 3

slow = nums[1] = 3，fast = nums[nums[3]] = nums[2] = 4
slow = nums[3] = 2，fast = nums[nums[4]] = nums[2] = 4
slow = nums[2] = 4，fast = nums[nums[4]] = nums[2] = 4

slow == fast，相遇在 4
```

寻找环入口：

```text
ptr = 0，slow = 4
ptr = nums[0] = 1，slow = nums[4] = 2
ptr = nums[1] = 3，slow = nums[2] = 4
ptr = nums[3] = 2，slow = nums[4] = 2

ptr == slow，相遇在 2
返回 2
```

### 示例 2

```text
nums = [3, 1, 3, 4, 2]

链表：
0 → 3 → 4 → 2
    ↑       ↓
    └───────┘

环的入口 = 3

返回 3
```

## 图解

```text
数组：[1, 3, 4, 2, 2]
索引： 0  1  2  3  4

链表结构：
0 → 1 → 3 → 2 → 4
            ↑   ↓
            └───┘

环的入口 = 2，也就是重复数字
```

## 复杂度分析

- 时间复杂度：`O(n)`
  - 快慢指针最多在线性步数内相遇，并找到环入口
- 空间复杂度：`O(1)`
  - 只使用常数个指针变量

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 数组转链表 | `i → nums[i]` 构成链表 |
| 必有环 | 长度为 `n + 1`，值范围是 `1 ~ n` |
| 环入口 | 重复数字对应的位置 |
| Floyd 判圈 | 快慢指针先找相遇点，再找环入口 |
| 不修改数组 | 只通过下标跳转，满足题目限制 |

## 为什么 `fast` 初始为 `nums[nums[0]]`？

如果让 `slow` 和 `fast` 都从同一个位置出发，并且先判断是否相等，就会一开始就相遇。

因此这里让：

```text
slow = nums[0]
fast = nums[nums[0]]
```

相当于慢指针先走一步，快指针先走两步，避免初始状态直接相遇，使后续循环正常执行。
