# 142. 环形链表 II

题目链接：[142. 环形链表 II](https://leetcode.cn/problems/linked-list-cycle-ii/description/)

## 思路

**快慢指针（Floyd 判圈算法）**

这道题要求找到链表开始入环的第一个节点，如果链表无环，则返回 `nullptr`。

可以分成两步：

1. 用快慢指针判断链表是否有环
   - 慢指针每次走 `1` 步
   - 快指针每次走 `2` 步
   - 如果有环，快慢指针一定会在环内相遇
2. 如果有环，再找到环的入口
   - 将其中一个指针重新放回 `head`
   - 另一个指针保持在相遇点
   - 两个指针每次都走 `1` 步
   - 再次相遇的位置就是环的入口

## 解题过程

1. 初始化：
   - `slow = head`
   - `fast = head`
2. 快慢指针找相遇点：
   - `slow` 每次走一步
   - `fast` 每次走两步
   - 如果 `fast == nullptr` 或 `fast->next == nullptr`，说明无环，返回 `nullptr`
   - 如果 `slow == fast`，说明有环，进入下一步
3. 找环入口：
   - 将 `slow` 移回 `head`
   - `fast` 保持在相遇点
   - 两个指针每次都走一步
   - 再次相遇的位置就是环入口
4. 返回环入口节点

## 代码

```cpp
class Solution {
public:
    ListNode *detectCycle(ListNode *head) {
        ListNode *slow = head;
        ListNode *fast = head;

        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;

            if (slow == fast) {
                slow = head;
                while (slow != fast) {
                    slow = slow->next;
                    fast = fast->next;
                }
                return slow;
            }
        }

        return nullptr;
    }
};
```

## 示例验证

### 示例 1

```text
链表：3 → 2 → 0 → -4
          ↑       ↓
          └───────┘

环入口是节点 2
```

快慢指针找相遇点：

```text
初始：slow = 3，fast = 3

slow = 2，fast = 0
slow = 0，fast = 2
slow = -4，fast = -4

slow == fast，相遇在 -4
```

寻找环入口：

```text
slow = head，也就是节点 3
fast = 相遇点，也就是节点 -4

slow = 2，fast = 2

再次相遇在节点 2
返回节点 2
```

### 示例 2

```text
链表：1 → 2
     ↑   ↓
     └───┘

环入口是节点 1
```

快慢指针最终会在环内相遇。

然后将一个指针放回 `head`，另一个指针留在相遇点，两者同步移动，最终会在节点 `1` 相遇。

### 示例 3

```text
链表：1 → nullptr

fast = 1
fast->next = nullptr

不满足循环条件，直接返回 nullptr
```

## 图解：为什么这样能找到环入口？

设：

```text
a = 从头节点到环入口的距离
b = 从环入口到相遇点的距离
c = 从相遇点绕环回到环入口的距离
```

链表结构可以抽象为：

```text
head ── a ──> 环入口 ── b ──> 相遇点
                ↑             │
                └──── c ──────┘
```

慢指针走过的距离是：

```text
a + b
```

快指针走过的距离是：

```text
a + b + k(b + c)
```

其中 `b + c` 是环的长度，`k` 表示快指针比慢指针多绕了几圈。

因为快指针速度是慢指针的 `2` 倍，所以：

```text
2(a + b) = a + b + k(b + c)
```

化简得到：

```text
a + b = k(b + c)
a = k(b + c) - b
a = (k - 1)(b + c) + c
```

这说明：

```text
从 head 到环入口的距离 a
=
从相遇点继续走到环入口的距离 c，再加上若干圈环长
```

所以让一个指针从 `head` 出发，另一个指针从相遇点出发，每次都走一步，它们一定会在环入口相遇。

## 复杂度分析

- 时间复杂度：`O(n)`
  - 快慢指针最多在线性步数内相遇，并找到环入口
- 空间复杂度：`O(1)`
  - 只使用常数个指针变量

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 快慢指针 | 判断链表是否有环 |
| 相遇条件 | `slow == fast` |
| 找入口 | 一个指针回到 `head`，另一个留在相遇点 |
| 同步移动 | 两个指针每次都走一步 |
| 无环情况 | `fast == nullptr` 或 `fast->next == nullptr` |
