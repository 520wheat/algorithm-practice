# 189. 轮转数组

题目链接：[189. 轮转数组](https://leetcode.cn/problems/rotate-array/description/)

## 思路

**三次反转**

向右轮转 `k` 位，等价于：

1. 反转整个数组
2. 反转前 `k` 个元素
3. 反转后 `n - k` 个元素

例如：

```text
nums = [1,2,3,4,5,6,7], k = 3

反转整个数组: [7,6,5,4,3,2,1]
反转前 3 个: [5,6,7,4,3,2,1]
反转后 4 个: [5,6,7,1,2,3,4]
```

这样就可以在原地完成数组轮转。

## 解题过程

1. 获取数组长度 `n`
2. 对 `k` 取模：`k %= n`，避免无效移动
3. 反转整个数组 `[0, n - 1]`
4. 反转前 `k` 个元素 `[0, k - 1]`
5. 反转后 `n - k` 个元素 `[k, n - 1]`

## 代码

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;

        reverse(nums.begin(), nums.end());
        reverse(nums.begin(), nums.begin() + k);
        reverse(nums.begin() + k, nums.end());
    }
};
```

## 示例验证

### 示例 1

```text
nums = [1,2,3,4,5,6,7], k = 3

reverse(nums.begin(), nums.end())
=> [7,6,5,4,3,2,1]

reverse(nums.begin(), nums.begin() + 3)
=> [5,6,7,4,3,2,1]

reverse(nums.begin() + 3, nums.end())
=> [5,6,7,1,2,3,4]
```

### 示例 2

```text
nums = [-1,-100,3,99], k = 2

反转全部：[-1,-100,3,99] -> [99,3,-100,-1]
反转前 2 个：[99,3,-100,-1] -> [3,99,-100,-1]
反转后 2 个：[3,99,-100,-1] -> [3,99,-1,-100]

结果为 [3,99,-1,-100]
```

## 其他解法

### 方法二：额外数组

用一个新数组保存每个元素轮转后的位置。

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;
        vector<int> tmp(n);

        for (int i = 0; i < n; i++) {
            tmp[(i + k) % n] = nums[i];
        }

        nums = tmp;
    }
};
```

### 方法三：环状替换

把每个元素直接放到它轮转后的位置，使用 `count` 记录已经移动的元素数量，避免重复移动。

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;
        int count = 0;

        for (int start = 0; count < n; start++) {
            int cur = start;
            int prev = nums[start];

            do {
                int next = (cur + k) % n;
                swap(nums[next], prev);
                cur = next;
                count++;
            } while (cur != start);
        }
    }
};
```

## 复杂度分析

- 时间复杂度：`O(n)`
  - 每个元素在反转过程中被访问常数次
- 空间复杂度：`O(1)`
  - 原地修改数组

## 关键点总结

| 步骤 | 说明 |
| --- | --- |
| 取模 | `k %= n`，避免无效移动 |
| 整体反转 | 先把后面的元素整体放到前面 |
| 局部反转 | 分别反转前 `k` 个和后 `n - k` 个元素，恢复内部顺序 |
| 原地操作 | 三次反转可以做到 `O(1)` 额外空间 |
